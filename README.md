# 毎月自動請求書PDF生成 → Gmail送信システム

> Googleスプレッドシートの顧客リストから請求書PDFを自動生成し、  
> Gmailで各顧客へ自動送付するGASツール。月初に自動実行。

---

## 機能

- スプレッドシートの請求リストを読み込み、**未送付の顧客へ一括送付**
- Googleドキュメントのテンプレートから**請求書を自動生成**
- 消費税を自動計算して合計金額を算出
- 生成した請求書を**PDF化**してGoogle Driveに自動保存
- PDFをGmailに添付して各顧客へ**自動送信**
- 送付後にスプレッドシートへ**送付済みフラグと日時を自動記録**
- 処理完了後に管理者へサマリーレポートをメール送信
- **毎月1日 09:00** にトリガー自動実行（設定後は完全ノータッチ）

## セットアップ手順

### 1. Googleドキュメントでテンプレートを作成

請求書レイアウトを作成し、以下のプレースホルダーを配置する：

```
{{顧客名}} {{請求番号}} {{請求年月}} {{請求日}} {{振込期限}}
{{サービス名}} {{金額}} {{消費税}} {{合計金額}}
```

### 2. スプレッドシートを作成

`createSampleSheet()` 関数を実行するとサンプルのシート構造を自動生成する。

### 3. スクリプトプロパティの設定

| プロパティ名 | 値 | 必須 |
|------------|-----|------|
| `INVOICE_TEMPLATE_ID` | テンプレートドキュメントID | 必須 |
| `INVOICE_SHEET_ID` | スプレッドシートID | 必須 |
| `OUTPUT_FOLDER_ID` | PDFの保存先DriveフォルダID | 任意 |
| `SENDER_NAME` | 差出人の表示名 | 任意 |
| `BANK_INFO` | 振込先情報 | 任意 |
| `TAX_RATE` | 消費税率（例: 0.10） | 任意 |

### 4. トリガーの設定

`setupMonthlyTrigger()` 関数を実行（初回のみ）。毎月1日 09:00 に自動実行するトリガーが設定される。

### 5. 動作テスト

`testRunSingle()` 関数を実行して、スプレッドシート2行目の1件のみテスト処理。

## こだわりポイント

### 完全自動化（設定後ゼロタッチ）
一度セットアップすれば毎月1日に自動実行。「請求書を送り忘れた」というヒューマンエラーを根絶。

### 二重送付防止
F列の「送付済みフラグ」で既送付レコードをスキップ。スクリプトを誤って2回実行しても同じ顧客に二重送付されない。

### PropertiesServiceによる認証管理
会社名・銀行口座など機密情報をスクリプトプロパティで管理。コードをGitHubに公開しても機密情報が漏洩しない。

### エラー耐性
1件の処理が失敗しても他の顧客への処理は継続。全件終了後にサマリーレポートで失敗件数・エラー内容を通知。
